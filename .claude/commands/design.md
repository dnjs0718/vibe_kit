# 디자인 단계

프로젝트의 디자인을 준비하는 단계입니다.

## 진행 방법

### 1단계: 디자인 파일 확인

유저에게 물어보세요:

"혹시 미리 준비한 디자인이 있으신가요? (피그마, 이미지 파일 등)"

### 경우 1: 디자인이 있는 경우

- 디자인 파일(이미지, 피그마 링크 등)을 공유해달라고 요청하세요.
- 공유받은 디자인을 분석하여 `project-brief.md`의 **디자인** 섹션에 추가하세요:

```markdown
## 디자인
- 디자인 참조: [링크 또는 파일 경로]
- 주요 컬러: [컬러]
- 레이아웃 특징: [설명]
```

### 경우 2: 디자인이 없는 경우

유저에게 두 가지 옵션을 제시하세요:

**옵션 A: AI 디자인 도구 활용**
"Google Stitch라는 도구를 사용하면 AI가 디자인을 만들어줍니다.
https://stitch.withgoogle.com/ 에 접속해서 원하는 화면을 설명해보세요.
완성된 디자인 스크린샷을 저에게 공유해주시면 반영하겠습니다."

**옵션 B: 알아서 만들어주세요**
"디자인은 제가 깔끔하게 만들어 드릴게요! 선호하는 스타일이 있으면 알려주세요."
- 선호 스타일 물어보기 (밝은/어두운, 미니멀/화려한, 참고 사이트 등)
- `project-brief.md`에 디자인 방향 기록:

```markdown
## 디자인
- 스타일: [유저가 선택한 스타일]
- 컬러 톤: [유저가 선호하는 톤]
- 참고: [있다면 참고 사이트]
- 방식: Claude가 자동 생성
```

### 2단계: 완료 안내

디자인 정보가 정리되면:
"디자인 준비가 완료되었습니다! 다음은 데이터베이스를 설정할 차례입니다."

그리고 `/setup-supabase` 커맨드를 실행하세요.
