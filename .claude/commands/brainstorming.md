# 기획 브레인스토밍

유저의 아이디어를 구체적인 기획으로 만드는 단계입니다.

## 진행 방법

### 1단계: 아이디어 파악

유저에게 다음을 물어보세요:

- "어떤 서비스를 만들고 싶으세요?"
- "이 서비스는 누가 사용하게 되나요?"
- "이 서비스가 해결하려는 문제가 뭔가요?"

유저가 뚜렷한 아이디어가 없다면:
- "평소에 불편하다고 느꼈던 것이 있나요?"
- "어떤 분야에 관심이 있으세요?"
와 같이 의도/목적부터 파악하세요.

### 2단계: 핵심 기능 정리

아이디어가 파악되면, 다음을 정리하세요:

- **서비스 이름** (가제)
- **한 줄 설명**
- **핵심 기능** (3~5개)
- **사용자 유형** (예: 일반 유저, 관리자)
- **주요 화면** (예: 홈, 로그인, 대시보드)

### 3단계: 유저 확인

정리한 내용을 유저에게 보여주고 확인받으세요.
수정할 부분이 있으면 반영하세요.

### 4단계: project-brief.md 저장

유저가 확인하면, 아래 형식으로 `project-brief.md` 파일을 프로젝트 루트에 저장하세요:

```markdown
# [서비스 이름]

## 한 줄 설명
[설명]

## 해결하려는 문제
[문제 설명]

## 대상 사용자
[사용자 설명]

## 핵심 기능
1. [기능 1]
2. [기능 2]
3. [기능 3]

## 주요 화면
1. [화면 1] - [설명]
2. [화면 2] - [설명]
3. [화면 3] - [설명]

## 기술 스택
- 프레임워크: Next.js (App Router)
- 데이터베이스/인증: Supabase
- 스타일링: Tailwind CSS
```

저장 후, 유저에게 "기획이 완성되었습니다! 다음은 디자인 단계입니다." 라고 안내하고 `/design` 커맨드를 실행하세요.
